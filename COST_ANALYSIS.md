# Gemini 3.6 Flash コスト分析

## 📊 正確な料金体系

### Gemini 3.6 Flash (`gemini-3.6-flash`)
- **Input:** $1.50 per 1M tokens ($0.0015 per 1K tokens) ※テキスト
- **Output:** $7.50 per 1M tokens ($0.0075 per 1K tokens)

> モデルは `api/config.ts` の `MODEL_NAME` でバージョン固定しています。
> エイリアス（`gemini-flash-latest`）は指す先と単価が予告なく変わるため使いません。
> `gemini-3.6-flash` は GA 版のため、モデルIDは安定しています（`-preview` サフィックスなし）。
> ⚠️ 3.6 Flash は「Flash-Lite」より上位ティアで、旧 3.5 Flash-Lite（$0.30/$2.50）より高単価です。

## 💰 典型的なリクエストのコスト計算

### シナリオ例
- **ユーザーメッセージ:** 200文字
- **会話履歴:** 5メッセージ (約500文字)
- **合計Input:** 700文字 ≈ 1,050トークン
- **AI応答Output:** 500トークン

### コスト計算
```
Input Cost  = 1,050 tokens × $0.0015 / 1K = $0.001575
Output Cost = 500 tokens × $0.0075 / 1K  = $0.003750
────────────────────────────────────────────────────
Total per request = $0.005325 (約 $0.0053)
```

## 🇯🇵 ¥5,000/月を予算にした場合の試算

> 前提: 為替 **1 USD = 163.19 JPY**（2026-07-22時点）→ ¥5,000/月 ≈ **$30.64/月**
> 1リクエスト = $0.005325（上記シナリオ）／ 月30日で平準化（¥166.7 ≈ $1.02/日）

**処理可能リクエスト数: $30.64 ÷ $0.005325 ≈ 約 5,750 リクエスト/月（≈ 192 リクエスト/日）**

### 1日あたりのアクティブ利用者数（1人あたりの利用量別）

| 1人あたり利用量 | 想定 | 収容人数/日 |
|----------------|------|------------|
| 30 メッセージ/日 | ヘビーユーザー | 約 6 人 |
| 20 メッセージ/日 | 積極的に利用 | 約 10 人 |
| 10 メッセージ/日 | 標準的な利用 | **約 19 人** |
| 5 メッセージ/日 | ライトユーザー | 約 38 人 |
| 3 メッセージ/日 | お試し | 約 64 人 |

> 1人1日10メッセージを毎日なら月300メッセージ = 約 $1.60（¥261）/人・月 → ¥5,000 で約 19 人。
> ✅ **月次上限を実装済み**：`api/rate-limiter.ts` に `maxCostPerMonth: 30.0`（≈¥5,000/月）を追加し、
> 月次コストを `cost:monthly:YYYY-MM` で集計。使い切ると `MONTHLY_COST_LIMIT` を返し翌月まで停止します。
> USD建てのため為替変動で¥換算は前後します（$30 ≈ ¥4,896 @¥163.19/USD）。
> ⚠️ 日次上限は $10 のまま。ヘビーな3日で月予算を使い切り得るため、平準化したい場合は
> `maxCostPerDay` も併せて下げてください（例: $1〜2）。

## 🚨 重要な注意事項

### 1. 無料枠の活用
Gemini APIには無料枠があります：
- Input/Output: 無料（制限あり）
- 適切に活用すれば初期コストは最小限

### 2. 生成パラメータの非推奨化
`api/chat.ts` の `temperature` / `topP` / `topK` は Gemini 3.6 時点で非推奨です。
現行モデルでは受理されますが、将来のモデル世代（Gemini 4+）では指定すると HTTP 400 に
なるため、モデル更新時に削除要否を確認してください。

### 3. コスト最適化の推奨事項

#### 即効性のある対策
1. **会話履歴の制限** (現在: 10メッセージ・1件8000文字上限)
   ```typescript
   const recentHistory = conversationHistory.slice(-5); // 10 → 5に削減
   ```
   効果: Input トークン ~50%削減

2. **出力トークン数の最適化** (現在: 4096)
   ```typescript
   maxOutputTokens: 2048, // 4096 → 2048に削減
   ```
   効果: 出力コスト最大50%削減（Output単価が高いため効果大）

3. **より安価なモデルへの切替**
   コスト最優先なら `gemini-3.5-flash-lite`（約0.29x）で約1/3以下に。
